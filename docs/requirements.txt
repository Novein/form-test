# 巡检无纸化系统（纯前端版）需求文档 v1

## 一、项目背景

目前工厂生产线巡检工作主要通过纸质表单进行记录，存在以下问题：

1. 纸质表单易丢失、损坏，不易长期保存
2. 手工填写效率低，容易漏填、错填
3. 数据无法快速汇总，统计与追溯困难
4. 表单模板调整需要重新印刷，成本高
5. 多人巡检记录汇总需要人工整理

为提升巡检记录效率与数据规范性，计划建设一套基于浏览器运行的巡检记录系统，实现巡检记录电子化、模板配置化、记录可导出与可汇总，从而逐步替代纸质巡检流程。

第一阶段系统采用**纯前端网站形式**实现，运行于 iPad 浏览器中，不依赖后端服务器与数据库。

---

# 二、项目目标

本系统的建设目标如下：

1. 实现巡检表电子化填写
2. 支持多名巡检人员轮流使用同一设备填写记录
3. 支持巡检模板、巡检项、结果选项通过配置文件管理
4. 支持巡检记录在本地浏览器中保存
5. 支持选择多条巡检记录合并生成 PDF 文件
6. 支持 PDF 下载并用于本地存档
7. 系统界面适配 iPad 触屏操作
8. 系统结构可扩展，便于未来接入后端系统

---

# 三、系统使用场景

系统使用方式如下：

1. 工厂现场配备一台或多台 iPad
2. 巡检工人轮流使用 iPad 打开巡检网站
3. 工人选择自己的姓名、班次、产线等信息
4. 按照巡检表逐项填写检查结果
5. 填写完成后提交记录
6. 当若干名工人完成巡检记录后
7. 由负责人选择这些记录并合并导出为 PDF
8. PDF 下载至本地或传输至存档系统

系统以“替代纸质巡检表”为核心目标。

---

# 四、系统范围（第一阶段）

第一阶段系统范围如下：

## 本阶段包含

* 配置驱动巡检模板
* 巡检记录填写
* 草稿自动保存
* 巡检记录本地存储
* 已完成记录管理
* 多条记录选择
* 合并导出 PDF
* iPad 浏览器适配

## 本阶段不包含

* 用户登录系统
* 权限管理
* 云端数据同步
* 后端数据库
* 图片上传
* 手写签名
* 审批流程
* 数据统计分析
* 报表系统
* 与其他系统集成

以上功能可在后续阶段扩展。

---

# 五、业务流程

系统整体业务流程如下：

```text
进入系统
   ↓
选择工人 / 日期 / 班次 / 产线
   ↓
进入巡检表
   ↓
填写巡检项
   ↓
暂存 或 提交
   ↓
记录保存到本地
   ↓
进入记录列表
   ↓
选择多条记录
   ↓
生成汇总
   ↓
导出 PDF
```

# 六、功能需求

## 6.1 配置文件管理

系统应通过配置文件定义巡检相关信息，而非写死在代码中。

配置文件应至少包含以下内容：

1. 巡检模板列表
2. 巡检项列表
3. 巡检结果类型
4. 巡检结果选项
5. 工人列表
6. 班次列表
7. 产线或区域列表
8. PDF 导出标题与格式信息

修改配置文件后，应无需修改系统核心代码即可调整巡检内容。

---

## 6.2 巡检记录填写

工人在填写巡检记录前需填写或选择基础信息，包括：

* 工人姓名
* 日期
* 班次
* 产线或区域
* 巡检模板

进入巡检页面后，系统根据配置文件生成巡检表单。

巡检项应支持以下类型（第一阶段建议支持）：

| 类型   | 示例              |
| ---- | --------------- |
| 单选   | 正常 / 异常         |
| 单选   | 已完成 / 未完成 / 不适用 |
| 数值输入 | 温度 / 压力 / 电流    |
| 文本备注 | 异常说明            |

系统应支持：

* 自动保存草稿
* 未填写项提示
* 提交前确认

---

## 6.3 草稿保存

在填写过程中系统应自动保存当前填写内容，防止误关闭页面导致数据丢失。

草稿状态的记录可以继续编辑，不计入已完成记录。

---

## 6.4 巡检记录提交

当工人填写完成后，可以提交巡检记录。

提交后的记录应：

* 保存到本地数据库
* 标记为已完成
* 出现在记录列表中
* 可查看、编辑、删除

每条记录至少包含以下信息：

* 记录ID
* 工人姓名
* 日期
* 班次
* 产线
* 模板名称
* 巡检项结果
* 备注
* 创建时间
* 更新时间
* 状态（草稿 / 已提交）

---

## 6.5 记录列表管理

系统应提供记录管理页面，用于查看当前设备中已保存的巡检记录。

列表页应支持：

* 按日期筛选
* 按工人筛选
* 按班次筛选
* 按产线筛选
* 查看记录详情
* 编辑记录
* 删除记录
* 勾选多条记录
* 批量操作（用于合并导出）

---

## 6.6 PDF 合并导出

系统应支持从记录列表中选择多条记录，将其合并生成一份 PDF 文件。

PDF 内容应至少包含：

* 报告标题
* 导出日期
* 巡检日期
* 工人信息
* 班次信息
* 产线信息
* 巡检项及结果
* 备注
* 导出时间

第一阶段 PDF 采用汇总报告形式，不要求完全还原纸质表格版式。

导出流程如下：

```text
选择记录
   ↓
生成汇总页面
   ↓
预览
   ↓
导出 PDF
   ↓
下载到本地
```

# 七、数据存储设计（纯前端）

由于第一阶段为纯前端系统，数据存储采用浏览器本地存储方式。

建议使用：

* IndexedDB 存储巡检记录
* localStorage 存储少量系统配置或用户偏好

数据存储内容包括：

* 草稿记录
* 已提交记录
* 系统配置缓存
* 导出记录标记

系统不保证跨设备同步，数据仅保存在当前设备浏览器中。

# 八、页面结构设计（建议）

系统建议包含以下页面：

| 页面    | 功能            |
| ----- | ------------- |
| 首页    | 功能入口          |
| 基础信息页 | 选择工人、班次、产线、模板 |
| 巡检填写页 | 填写巡检表         |
| 记录列表页 | 查看与管理记录       |
| 记录详情页 | 查看或编辑记录       |
| 导出预览页 | 合并记录并导出 PDF   |
| 配置说明页 | 模板与配置说明       |

核心页面不宜过多，以便现场人员使用简单。

# 九、非功能需求

## 9.1 设备适配

系统需适配 iPad 浏览器，包括：

* 触屏操作
* 按钮尺寸适合手指点击
* 支持横屏与竖屏
* 页面滚动流畅

## 9.2 易用性

由于使用人员为现场工人，系统应：

* 界面简洁
* 操作步骤少
* 字体较大
* 提示清晰
* 错误提示明显

## 9.3 稳定性

系统应具备：

* 自动保存
* 页面刷新恢复草稿
* 提交确认
* 防止重复提交

## 9.4 可扩展性

系统设计应支持未来扩展：

* 后端数据库
* 用户登录
* 权限管理
* 图片上传
* 电子签名
* 数据统计
* 报表系统

# 十、系统边界（非常重要）

本系统第一阶段定位为：

**单设备、纯前端、巡检记录填写与 PDF 导出工具**

系统不解决以下问题：

* 多设备数据同步
* 用户权限管理
* 数据长期集中存储
* 审批流程
* 报表分析
* ERP/MES 集成

这些属于后续阶段系统升级范围。

---

# 十一、待确认问题（建议你后续和现场一起确认）

下面这些问题建议你和实际使用人员沟通确认：

1. 是否只有一台 iPad 使用，还是多台设备
2. 巡检模板是否有多套
3. 巡检项是否包含数值、备注、照片等类型
4. PDF 是否需要完全还原纸质表单格式
5. 一份 PDF 对应多少人的记录，是否由用户手动选择
6. 是否需要记录填写时间与提交时间
7. 提交后记录是否允许修改
8. 本地记录需要保留多少天
9. 是否需要导出 Excel（可能后面会需要）
10. 是否需要记录异常项汇总

这些问题会影响后续系统结构设计。

# 十二、建议的第一阶段目标总结
